因端口映射造成Nginx重定向异常

发表于 2022-09-23 本文字数 977 字 阅读时长 1 分钟

首页 异常修复

# 【问题描述】

  1. nginx 访问带目录的 url 时（例： admin )，若未在目录最后添加 / ，ngxin 则会进行一次默认的重定向（即： ip:port/admin 重定向为 ip:port/admin/ ）

  2. 该重重定向默认为绝对路径的重定向模式（在跳转中可以查看响应头 (Response Headers) 中的 Location 处看到完整的 url 链接确定）。

  3. 但 nginx 对应的此绝对路径则会将对应的端口切换为其靶机内部所使用的端口。

  4. 因此，若进行了端口映射的情况下，通过指定端口对靶机进行了一次请求，因 nginx 自身默认的绝对路径重定向导致相应的 url 端口进行了变动（由外网映射端口变更为内网端口）。则会造成 url 寻址错误。

# 【特征点】

  1. 存在一次 字符串 至 字符串/ 的重定向请求
  2. 响应头为 ip:port/字符串/ 的绝对路径，且此处端口与请求 url 对应端口不一致

# 【解决方法】

此处需要关闭 nginx 默认开启的绝对重定向。即在该网页对应的 conf 文件中的 server 中添加 absolute_redirect off; ，或者在 nginx.conf 文件中的 http 中添加 absolute_redirect off; 对该服务器中所有的网页服务均关闭绝对重定向。

  1. 确定当前运行的 nginx 路径： ps aux | grep nginx （一般不需要此步骤）

  2. 查找到 nginx 对应配置文件： /usr/sbin/nginx -V

    1. 此处对应第一步的 nginx 对应路径
    2. 一般直接跳过步骤 1，运行 nginx -V 即可
    3. 查找 --conf-path=xxxxx ，这里 xxx 处对应的即为 nginx 配置文件路径

  3. 此处将提出两种解决方法，分别为：关闭整个服务器中 nginx 重定向；关闭指定网页的 nginx 重定向

    1. 关闭整个服务器中 nginx 重定向

      修改步骤 2 中查到的 etc/nginx/nginx.conf 文件，在其 http 中添加 absolute_redirect off; 并重启 nginx 即可对该服务器中所有的网页服务均关闭绝对重定向，问题解决（可正常使用）。

    2. 关闭指定网页的 nginx 重定向

      1. 进入 etc/nginx/nginx.conf 文件，查找 http 下的 include ，确定网页对应的配置文件路径

      2. 找到指定网页的配置文件，在其 server 中添加 absolute_redirect off; 并重启 nginx 即可对该网页服务关闭绝对重定向，问题解决（可正常使用）。
